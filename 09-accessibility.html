K-Sutda — 접근성 (Accessibility) 가이드

색맹 / 시각 / 청각 / 운동 / 인지 접근성. 글로벌 출시 (Apple / Google 스토어 심사 통과) + 윤리적 게임 디자인.

0. 접근성 원칙

0-1. 왜 중요한가

  • 한국 사용자 중 색맹 약 6% (한국 남성), 저시력 노인 증가
  • 글로벌 출시 시 EU EAA (European Accessibility Act, 2025) 영향
  • Apple App Store / Google Play 의 접근성 가이드 준수 권장
  • 윤리적 디자인 — 더 많은 사용자가 즐길 수 있도록

0-2. WCAG 2.1 기준 (적용 가능 범위)

본 게임이 100% WCAG AA 준수를 목표로 하지는 않지만, 핵심 사용자 경험 차단 없도록 다음 항목 검토:

WCAG 항목 적용
1.4.3 명도 대비 (4.5:1) 텍스트 / 배경
1.4.11 비텍스트 대비 (3:1) 버튼 / 카드 보더
2.1.1 키보드 접근 모든 인터랙션 키보드 가능 (미래)
2.4.4 링크 목적 버튼 텍스트 명확
2.5.5 터치 타겟 (44×44) 모바일
3.3.1 입력 에러 식별 닉네임 등
4.1.3 상태 메시지 토스트 / 라이브 영역

1. 색 / 명도 (Color & Contrast)

1-1. Palette 명도 대비 검증

현재 Palette 조합의 명도 대비 (WCAG 기준):

조합 대비비 WCAG 등급
TEXT_PRIMARY (#F2F2F2) on BG_INK (#0D2619) 13.5 : 1 ✅ AAA
TEXT_PRIMARY on BG_EARTH (#131313) 14.2 : 1 ✅ AAA
TEXT_PRIMARY on BG_DARK_BLUE (#15191F) 13.8 : 1 ✅ AAA
TEXT_SECONDARY (#A6A6B2) on BG_INK 6.1 : 1 ✅ AA
TEXT_MUTED (#666673) on BG_INK 3.2 : 1 ⚠ 작은 텍스트 부적합
ACCENT_GOLD (#F3C72E) on BG_INK 9.4 : 1 ✅ AAA
ACCENT_VERMILION (#D93333) on BG_INK 4.8 : 1 ✅ AA
ACCENT_TEAL (#33A68C) on BG_INK 5.7 : 1 ✅ AA

규칙:

  • TEXT_MUTED 는 12px 이하 보조 텍스트만 사용 (가독성 필수 영역 X)
  • 액센트 색 (금/주홍/청록) 모두 AA 통과 — 적극 사용 가능

1-2. 색맹 대응

한국 남성 색맹 비율:

  • 적록색맹 (deuteranopia, protanopia): 약 5~6%
  • 청황색맹 (tritanopia): 약 1%

문제 가능 조합 + 대안:

의미 현재 색맹 시 위험 대안
위험 (빨강) vs 정상 주홍 vs 흰색 적록 → 잘 보임 ✅ 그대로 OK
1차 (주홍) vs 2차 (금) 주홍 vs 금 적록 → 둘 다 비슷한 갈색 ⚠ 아이콘 추가 (🃏 vs 🎴) 이미 적용
광패 강조 (금) 금색 보더 적록 → 회색에 가까움 ⚠ 보더 두께 차이 (2px → 4px) + 글로우
희귀도 (4단계) 회/청/자주/주황 적록 → 청·자주 구분 어려움 ⚠ 아이콘 추가 ★, ★★, ★★★, ★★★★

1-3. 다크모드 / 라이트모드

  • 다크모드만 지원 (기본 — 한국 도박판 톤)
  • 라이트모드 미지원 (작업 우선순위 낮음)
  • HTML 자료 (design-handoff/) 는 OS 다크/라이트 모두 자동 대응

1-4. 디자이너 검증 도구

  • Figma 플러그인: "Color Blind" / "Contrast" — 색맹 시뮬레이션
  • Stark (Figma/Sketch) — WCAG 대비 자동 검증
  • Chrome DevTools — Rendering tab → Emulate vision deficiencies

2. 폰트 / 가독성

2-1. 최소 폰트 사이즈

종류 최소 권장
본문 (BODY) 14px 16px
보조 (CAPTION) 10px 12px
모바일 (작은 화면) 16px 18px

→ 현재 Palette.FONT_CAPTION = 12px 적절. FONT_BODY = 16px 적절.

2-2. 폰트 선택

  • 한글: Pretendard, KoPub바탕체, NotoSansKR — 모두 가독성 검증된 폰트
  • 영문 fallback: Inter, Segoe UI, Apple SD Gothic Neo
  • ❌ 손글씨체 / 디스플레이 폰트 — 본문 사용 금지 (디자인 강조 카피만)

2-3. 글자 간격 (line-height)

  • 본문: line-height 1.5~1.7 (한글 가독성 위해)
  • 버튼: line-height 1.2 (단일 행)
  • 코드 / 숫자: line-height 1.4

2-4. 텍스트 줄바꿈

  • 한글: word-break: keep-all (단어 단위 줄바꿈, 가독성 ↑)
  • 긴 영문 단어: overflow-wrap: anywhere
  • design-handoff/ HTML 변환기 (scripts/tools/md_to_html.sh) 에 이미 적용됨

3. 모션 / 애니메이션

3-1. Reduced Motion 옵션 (권장 — 미래)

WCAG 2.3.3 — 사용자가 모션 비활성화 가능해야.

Settings → Reduced Motion: [ON/OFF]
  ON 시:
    - 카드 fade-in/flip → instant
    - 점수 폭발 → scale 없이 fade-only
    - 위험 영역 펄스 → 색만 변경
    - scene transition → 즉시

OS 시스템 설정 자동 감지:

  • Web: prefers-reduced-motion: reduce
  • iOS/Android: 시스템 reduced motion 설정

→ Phase 4 별도 sprint.

3-2. 광민감 (Photosensitive) 고려

  • 빠른 깜빡임 ❌ (초당 3회 이상 ❌)
  • 위험 영역 펄스: 0.45s 1회 (안전)
  • 임박 펄스 (제안): 0.6s loop — 초당 1.67회, 안전 범위
  • 점수 폭발: 1회만, 반복 없음

→ WCAG 2.3.1 (3회/초 미만) 준수.

4. 청각 접근성

4-1. 자막 / 대안 텍스트 (필수)

모든 사운드 효과는 시각 대안 제공:

사운드 시각 대안
card_flip 카드 뒤집기 시각 (이미 보임)
card_deal 카드 슬라이드인 (이미 보임)
win 점수 폭발 라벨
lose 결과 라벨 "[패배]"
button_click 버튼 pressed 상태
mapa_trigger 슬롯 펄스 + last_triggered 라벨
first_dia "AI 다이!" 토스트
bluff_success "🃏 블러프 성공!" 토스트

→ 청각 장애 사용자도 정보 손실 없음 ✅

4-2. 사운드 옵션 (권장 — 미래)

Settings →
  Master Volume    [────|─────] 60%
  Music (BGM)      [────|─────] 50%
  Sound Effects    [────|─────] 80%
  [✓] BGM 사용
  [✓] SE 사용

4-3. BGM / SE 분리

  • BGM 끄기 가능 (집중력 향상 / 청각 민감 사용자)
  • SE 는 게임 정보 전달 — 끄기 가능하지만 권장 X

5. 운동 (Motor) 접근성

5-1. 터치 타겟 (모바일)

종류 최소 현재
일반 버튼 44×44 px (iOS) / 48×48 dp (Android) 130×64 — ✅
카드 (mulligan 트리거) 동일 220×320 — ✅ 충분
슬라이더 핸들 44×44 (확인 필요)

→ 현재 데스크탑 UI 는 충분히 크지만, 모바일 적응 시 재확인 (13 가이드 참조).

5-2. 버튼 간격

  • 최소 8px (실수 클릭 방지)
  • 위험한 액션 (다이 / 폴드) 은 안전 액션 (콜) 과 멀리

5-3. 키보드 전용 사용자 (PC)

  • 모든 액션 키보드 가능해야 (마우스 없이도 플레이)
  • Tab 으로 버튼 순환 + Enter 로 선택
  • 시각 focus indicator (보더 강조) 필수

→ Phase 4 별도 sprint.

5-4. 시간 제한 없음

  • K-Sutda 는 턴 베이스 — 사용자가 충분히 생각 가능
  • 자동 진행 / 타이머 없음 (도박 게임의 압박감 회피 + 접근성)

6. 인지 접근성

6-1. 명확한 정보 위계

8번 가이드 §6 참조 — 우선순위에 따라 시각 강도 차이.

6-2. 일관성 (Consistency)

  • 모든 화면에서 같은 색 = 같은 의미 (주홍 = 위험 / 1차, 금 = 강조 / 2차)
  • 같은 버튼 위치 (BackButton 항상 하단 중앙)
  • 같은 모션 (M-XX 명세 따름)

6-3. 학습 곡선

  • 첫 런 튜토리얼 (HelpInfo + Board 인라인 안내) — 8번 §11 참조
  • 게임 규칙 항상 접근 가능 (HelpInfo 메뉴)
  • 카드 효과 hover tooltip (Sprint 3.8 — 이미 구현)

6-4. 에러 방지 / 복구

  • 자동 세이브 (매 라운드) → 게임오버 후 재개
  • 결정성 (시드 기반) — 같은 시드로 재시도 가능 (디버그 / 학습 용도)

7. 한국어 / 영어 접근성 차이

항목 한국어 영어
폰트 사이즈 16px 가독성 OK 14px 도 OK (영문 좁음)
줄바꿈 word-break: keep-all word-break: normal
폰트 fallback Pretendard → NotoSansKR → SystemKR Inter → SF → Segoe
글자 길이 짧음 ("다이") 길음 ("Fold") — 1.5배

→ localization 시 길이 변화 대응 (10 가이드 참조).

8. 플랫폼별 접근성 가이드

8-1. Apple (iOS / macOS)

  • HIG (Human Interface Guidelines) 접근성 챕터 준수
  • VoiceOver 지원 (모든 UI 요소에 accessibility label)
  • Dynamic Type 지원 권장 (시스템 폰트 사이즈 따름)

→ Phase 4 별도 sprint.

8-2. Google (Android)

  • Material Design Accessibility 가이드
  • TalkBack 지원
  • 시스템 다크모드 자동 (현재 다크만 지원이라 이슈 없음)

8-3. Steam / PC

  • Steam Deck 컨트롤러 지원 (D-pad + A/B) — Phase 4
  • Big Picture 모드 호환 (큰 화면 UI scaling) — 현재 1280×720 기준, 4K 까지 자동 스케일

9. 베타 테스트 — 접근성 항목

베타 사용자 모집 시 다음 그룹 의도적 포함:

  • 색맹 사용자 1~3명
  • 저시력 노인 사용자 1~2명
  • 청각 장애 사용자 1명 (자막 / 시각 대안 검증)
  • 키보드 전용 사용자 1명 (Phase 4 후)

피드백 채널에 "접근성" 태그 별도 운영.

10. 작업 우선순위

Tier 1 — 즉시 (Sprint 3.15 이내)

  • Palette 명도 대비 검증 (위 §1-1)
  • 폰트 사이즈 최소값 확인 (12px 이하 보조만)
  • 청각 대안 — 모든 사운드에 시각 동반 (현재 만족)

Tier 2 — 베타 직후 (Sprint 4.x)

  • 사운드 옵션 화면 (BGM/SE 볼륨)
  • Reduced Motion 옵션
  • 키보드 단축키 (1/2/3 = 콜/레이즈/다이)
  • 색맹 시뮬레이션 검증 (디자이너 + 베타 사용자)

Tier 3 — 모바일 / 글로벌 출시

  • iOS VoiceOver 라벨
  • Android TalkBack 라벨
  • Steam Deck 컨트롤러
  • Dynamic Type 지원

11. 라이선스 / 컴플라이언스

  • 글로벌 출시 시 WCAG 2.1 AA 자체 검증 보고서 첨부 권장 (Apple/Google 심사)
  • EU 시장 진출 시 EAA 2025 준수 확인
  • 한국: 장애인차별금지법 (디지털 접근성) — 게임은 직접 의무 대상 아니지만 권장

12. 디자이너 협업 체크리스트

새 화면 시안 만들 때:

  • 텍스트 / 배경 명도 대비 ≥ 4.5:1 (자동 검증 도구)
  • 색만으로 정보 전달 X (아이콘 / 텍스트 보조)
  • 폰트 14px 이상 (보조 12px 이상)
  • 클릭 가능 영역 44×44 이상 (모바일 고려)
  • 모든 사운드 효과에 시각 대응
  • 빠른 깜빡임 / 강한 플래시 ❌
  • focus indicator (키보드 사용 시 보더 강조)
  • 빈 상태 / 에러 상태 명확한 안내
  • 한글 word-break: keep-all 적용
