K-Sutda — UI/UX 가이드라인

인터랙션 패턴 / 상태 표현 / 피드백 / 마이크로카피 일관성 가이드. 디자이너가 시안 만들 때 매번 참고.

0. 핵심 원칙

0-1. 사용자 의사결정 무게감

K-Sutda 의 본질은 카드 1장씩 보고 선택. UI는 그 결단을 도와주는 도구이지, 화려한 장식이 아님.

  • 명확성 우선 — 베팅 금액·명성치 잔액·1차/2차 단계 즉시 인지 가능
  • 결정 단계 차단 — 클릭 후 0.5초 내 시각·청각 피드백 필수
  • 과한 모션 회피 — 점수 폭발 등 임팩트는 정해진 순간만, 그 외엔 차분

0-2. 한국 도박판 톤

  • 차분한 진중함 + 빠른 반응
  • 발라트로 같은 톡톡 튀는 톤 ❌ → 무게감 + 정밀
  • 마이크로카피도 한국적: "한 판 더" / "다이" / "운이 좋았군"

1. 버튼 인터랙션

1-1. 표준 상태 (4가지)

모든 버튼은 4 상태를 시각으로 명확히 구분:

상태 시각 모션 (M-11 참조)
default 한지 베이지 베이스 + 먹색 텍스트 —
hover 살짝 밝아짐 + 보더 진해짐 scale 1.0 → 1.03 (0.1s)
pressed 색 어두워짐 + 텍스트 살짝 작아짐 scale → 0.97 → 1.0 (0.15s)
disabled alpha 0.5 + 보더 회색 —

1-2. 버튼 종류별 가이드

종류 색 사용처
Primary 베이지 / 먹 텍스트 "새 런 시작" / "콜"
Danger 주홍 보더 / 주홍 텍스트 "다이" / "삭제"
Success 금색 보더 / 금 텍스트 "1차 레이즈" (Sprint 3.13 톤)
Secondary 회색 보더 / 회색 텍스트 "취소" / "메인 메뉴로"
Ghost 보더만 + 투명 배경 "새로고침"

1-3. 1차 vs 2차 베팅 버튼 (Sprint 3.13 — 확정)

1차 단계:
  ┌────────┐ ┌──────────┐ ┌────────┐
  │콜(1차5)│ │1차레이즈+5│ │다이(-5)│  ← 모두 주홍 톤
  └────────┘ └──────────┘ └────────┘

2차 단계:
  ┌────┐ ┌──────┐ ┌────┐ ┌────┐
  │ 콜 │ │레이즈│ │올인│ │폴드│  ← 금색 톤
  └────┘ └──────┘ └────┘ └────┘

→ 사용자가 단계 인지 부담 ↓.

1-4. 클릭 피드백 (필수)

  • 시각: scale pulse (1.0 → 0.97 → 1.0)
  • 청각: button_click SE
  • 상태 변경: 즉시 (disabled / loading 등)

2. 입력 / 폼

2-1. 입력 필드 (LineEdit)

현재 위치: MainMenu 닉네임 입력

상태 시각
default 한지 베이지 배경 + 옅은 보더
focus 보더 금색 + 살짝 글로우
error 보더 주홍 + 하단 작은 에러 텍스트
success 보더 청록 (저장 완료 시)

2-2. 슬라이더 (RaiseSlider)

상태 시각
default 한지 베이지 트랙 + 금색 핸들
active 트랙 진해짐 + 핸들 +scale
value 표시 슬라이더 옆 큰 라벨 ("레이즈: 30")

2-3. 입력 검증 마이크로카피

상황 카피
닉네임 빈 값 "닉네임을 입력하세요" (placeholder)
닉네임 너무 김 "20자 이내로 입력해 주세요" (자동 자름 + 알림)
닉네임 저장 "닉네임 저장: <이름>" (status 라벨, 청록)
명성치 부족 (1차 레이즈) "명성치가 부족합니다" (토스트)
잔액 부족 → all_in 자동 폴백 "잔액 부족으로 올인 처리" (토스트, 회색)

3. 상태 표시

3-1. 로딩 상태

종류 시각
단순 로딩 (leaderboard fetch) 텍스트 "불러오는 중..." + ⏳ 아이콘 회전
저장 중 (autosave) 우상단 작은 ✓ 페이드 인/아웃 (1초)
전환 중 (scene change) M-10 검은 페이드 (0.5s)

3-2. 빈 상태 (empty state)

화면 빈 상태 카피
Leaderboard (기록 없음) "(아직 기록 없음 — 첫 런을 완료해 보세요)"
DailyActivity (기록 없음) "(기록 없음)" — 카드 안에, 회색
DailyActivity (안 플레이 한 날) "안 플레이" — 일자별 row, 회색
Market (구매 가능 카드 없음) (이런 상황 거의 없음 — 4 카드 항상 진열)

3-3. 에러 상태

상황 카피 + UI
세이브 버전 불일치 "세이브 버전 불일치 — 새 런 시작 권장" (status 라벨)
저장된 런 없음 "저장된 런이 없습니다" (continue 버튼 비활성 + tooltip)
Supabase 오프라인 "📁 로컬 기록 (Supabase 키 미설정 — 오프라인 모드)" (회색)

4. 토스트 / 알림

4-1. 표준 토스트 (M-09 참조)

┌──────────────────────────────┐
│ 🃏 블러프 성공!  +1.5 mult    │
└──────────────────────────────┘
화면 상단 중앙. 3.25s 후 자동 사라짐.

4-2. 토스트 종류

종류 색 아이콘 예시
info 회색 + 흰 텍스트 ℹ️ "재배분 완료"
success 금 + 먹 텍스트 🏆 "단계 3 클리어!"
warning 주홍 + 흰 텍스트 ⚠ "명성치 위험 영역 진입"
danger 짙은 주홍 + 흰 텍스트 ⚠️ "게임오버 임박"
bluff 금 + 먹 텍스트 🃏 "블러프 성공!"

4-3. 토스트 위치

  • 기본: 화면 상단 중앙 (margin-top 60px)
  • 게임 중요 알림: 카드 영역 위쪽 (가독성)
  • 동시 다발 토스트: 위에서 아래로 stack (max 3, 초과 시 가장 오래된 것 자동 제거)

5. 모달 / 다이얼로그

5-1. 모달 사용 시점 (현재 없음 — 미래)

  • Soul 카드 발견 (Legendary 획득)
  • 단계 클리어 시 다음 단계 진입 확인 (옵션)
  • 옵션 메뉴 (BGM/SE 볼륨)
  • 종료 확인 ("정말 메인 메뉴로?")

5-2. 모달 구조

┌──────────────────────────────────┐
│  [반투명 검은 오버레이]          │
│                                  │
│    ┌──────────────────────┐      │
│    │ 🏆 Soul 카드 발견!   │      │ ← 헤더 (24px Bold)
│    ├──────────────────────┤      │
│    │                      │      │
│    │  무한 배수            │      │ ← 본문
│    │  (Legendary)          │      │
│    │  "매 T6 마다 누적     │      │
│    │   ×1.05 영구 복리"   │      │
│    │                      │      │
│    ├──────────────────────┤      │
│    │   [획득]  [넘기기]   │      │ ← 버튼
│    └──────────────────────┘      │
└──────────────────────────────────┘
배경: 검은 오버레이 alpha 0.7

규칙:

  • ESC 키 = 취소 (또는 우측 X)
  • 외부 영역 클릭 = 취소 (또는 무시 — 화면별 결정)
  • 모달 등장 모션: alpha + scale 0.9 → 1.0 (0.2s, BACK)

6. 정보 위계 (Information Hierarchy)

6-1. Board 화면 우선순위

가장 중요한 정보를 가장 큰 사이즈·강한 색으로:

우선순위 정보 시각
🔴 1 명성치 24px Bold, 위험 시 빨강 펄스
🔴 2 player 카드 2장 220×320 (가장 큰 시각)
🟡 3 결과 라벨 (승/패/breakdown) 16px, 색 구분 (승=금/패=회색)
🟡 4 판돈 / commit 분리 16px, 강조 X
🟢 5 마패 슬롯 (3개) 작은 카드 + 발동 시 펄스
🟢 6 단계 / 라운드 / AI 패턴 12~14px, 상단 메타
🟢 7 deck_tracker (사용 카드 수) 12px, 상단 작은

6-2. 폰트 위계 (Palette.gd 와 일관)

TITLE    96px   MainMenu / 점수 폭발 / 게임오버 큰 라벨
HEADING  32px   섹션 헤더 / 모달 제목
SUBHEAD  24px   토스트 / 카드 컴포넌트 헤더
BODY     16px   일반 본문 / 버튼 / 결과 라벨
CAPTION  12px   보조 정보 / 시드 / 메타

7. 마이크로카피 (Microcopy)

7-1. 톤 원칙

  • ✅ 짧고 결단적: "한 판 더", "다이", "콜"
  • ✅ 한국적 어휘: "광 한 번", "운이 좋군", "전설"
  • ❌ 영어 직역 ("게임 시작!", "확인", "성공!") — 한국적이지 않음
  • ❌ 도박 미화 ("대박!", "잭팟!", "이긴다!") — 사행성 회피
  • ❌ 코미디 ("앗! 망통이라니!") — 톤 안 맞음

7-2. 핵심 어휘 표

영어 / 일반 K-Sutda 어휘
Score 명성치 (名聲)
Bet 베팅 / 콜 / 레이즈
Fold 다이 (1차) / 폴드 (2차)
Win 승리 / 광 보임
Lose 패배 / 패 보임
Run 런 (한 번의 도전)
Stage 단계
Round 판
Joker 마패 (タッザ)
Hand 패
Shop 시장
Bluff 뻥 / 블러프
Game Over 게임오버

7-3. 결과 라벨 (현재 코드 기준)

[승리] (공개)  상대 jokbo=30 (점수 90) / 내 jokbo=100 (점수 1500)   명성치 +1500  🃏 블러프!
[패배] (다이) ...
[무승부] ...

→ 디자이너가 시안 그릴 때 위 형식 그대로 사용.

8. 마우스 / 키보드 / 터치

8-1. 마우스 (PC)

액션 행동
Left Click 기본 액션 (버튼 / 카드)
Right Click (없음 — 미래: 카드 자세히 보기?)
Hover 버튼 / 카드 hover 상태

8-2. 키보드 (미래)

키 액션
Space / Enter 기본 (Confirm)
ESC 메인 메뉴로 / 모달 닫기
1 / 2 / 3 1차 콜 / 레이즈 / 다이 (옵션)
C / R / F / A 콜 / 레이즈 / 폴드 / 올인 (영문 단축키)

→ Phase 4 별도 sprint.

8-3. 터치 (모바일 — 13 적응 가이드 참조)

  • 최소 터치 타겟: 44 × 44 px (Apple HIG) / 48 × 48 dp (Material)
  • 버튼 간 간격: 최소 8 px
  • 카드 클릭 = mulligan: 카드 자체가 충분히 큼 (220×320)

9. 실수 방지 (Error Prevention)

9-1. 위험한 액션은 확인

액션 확인
새 런 시작 (기존 세이브 덮어쓰기) (현재 없음 — 추가 권장: "기존 세이브가 삭제됩니다. 계속?")
명성치 한도 직전 all_in (현재 없음 — 추가 권장: "잔액 전체 베팅합니다. 다이가 더 안전합니다")

9-2. Undo / Redo (게임 디자인 결정)

  • 결정성 게임: 베팅 후 undo 없음 (도박 본질)
  • 단, mulligan 은 명성치 차감 — 한 번 더 클릭하면 다시 새 카드 (8명성치 추가)
  • 게임오버 전: 자동 세이브 → 게임오버 후 "다시 시작" 가능

10. 일관성 체크리스트 (디자이너 self-check)

새 화면 / 컴포넌트 시안 만들 때:

  • Palette 색만 사용? (커스텀 색 X)
  • 폰트 위계 5단계 중 1 선택?
  • 8px 그리드 (XS/S/M/L/XL spacing 사용)?
  • 버튼 4 상태 (default/hover/pressed/disabled) 모두 시안?
  • 빈 상태 (empty) 카피 + 시안?
  • 에러 상태 + 메시지?
  • 모션 표 (04-motion-spec) 의 사양 준수?
  • 마이크로카피 톤 — 한국적, 결단적, 사행성 회피?
  • 1차 vs 2차 베팅 색 구분 (주홍 vs 금) 유지?
  • 명성치 위험 영역 (< 0 / < -100) 빨강 / 펄스?

11. 인게임 가이드 / 튜토리얼

11-1. 현재 (Sprint 3.15)

  • HelpInfo 화면 (메인 메뉴 "📖 게임 규칙") — 정적 RichText
  • Board 진입 시 별도 튜토리얼 없음

11-2. 권장 추가 (베타 후)

  • 첫 런 튜토리얼: Board 진입 첫 라운드에 "1차 콜 (5명성치)" 버튼 강조 + 화살표 (3초 후 fade-out)
  • 첫 mulligan 안내: 두 번째 카드 받은 직후 "카드를 클릭하면 8명성치로 재배분 가능" 한 번만
  • 블러프 첫 성공: 첫 블러프 성공 시 모달 "블러프 시스템 활성화! 약한 패 + raise → AI 다이 유도하면 추가 점수"
  • 단계 8 첫 도달: "마지막 보스, 전설의 마패 남궁천 등장" (full screen intro)

12. 외부 디자이너 협업 권장

12-1. Figma 활용

  • 각 화면 별 frame + 상태 (default/hover/empty/error) 모두 그리기
  • 컴포넌트 라이브러리 (11번 자료 참조) 를 Figma component 로 등록
  • 디자인 변경 시 PR 형식으로 개발자 리뷰

12-2. 디자이너 → 개발자 핸드오프

  • Figma "Dev Mode" 활용 (CSS / SVG 값 자동 추출)
  • 모션은 Lottie .json 또는 .mp4 reference
  • 색은 hex 명시 (Palette.gd 와 매칭 확인)
