K-Sutda — 모션 명세서

게임 내 모든 애니메이션 / 트랜지션의 사양. After Effects / Lottie 작업, 또는 Godot Tween 튜닝 시 참고.

0. 기본 원칙

0-1. 모션 톤

  • 무게감 + 빠른 반응 — 한국 전통 도박판의 진중함 + 게임으로서의 즉각성
  • 시간 단위 표기: 모두 초 (seconds) 사용. ms 가 아닌 0.15s, 0.2s 등.
  • 이징 (easing): 기본 SINE_OUT / 임팩트 BACK_OUT / 진정 QUAD_OUT
  • 연속 모션 분해: 각 단계 0.15s ~ 0.4s. 1초 넘는 모션 피하기 (지루함).

0-2. 60 FPS 가정

모든 timing 은 60 FPS 기준 부드러움. 30 FPS 환경에서도 자연스러워야.

0-3. Reduced Motion 옵션 (선택)

향후 접근성 옵션 — 모든 모션 비활성화 / 단순 fade 로 대체. 본 sprint 스코프 외.

1-4. (제안 — 신규) 카드 mulligan 교체

두 번째 카드를 클릭 → 8명성치 차감 → 새 카드 등장.

단계 모션
1. 클릭 피드백 카드 scale 1.0 → 0.95 → 1.0 (0.1s, BACK)
2. flip out scale.x 1.0 → 0.0 (0.15s, SINE_IN)
3. 카드 교체 callback (instant)
4. flip in scale.x 0.0 → 1.0 (0.15s, SINE_OUT)
5. 글로우 1초간 금색 보더 페이드 (0.5s in, 0.5s out)

총 0.40s + 1.0s glow. 사용자가 "내가 바꿨다" 명확 인지.

2-3. 위험 영역 펄스 (Board.gd._refresh_prestige_warning — Sprint 3.14)

명성치 < 0 (위험) / < -100 (임박) 시 prestige_label 펄스.

속성 값
위험 (< 0) scale 1.0 → 1.05 → 1.0 (0.2s + 0.25s, SINE)
임박 (< -100) scale 1.0 → 1.15 → 1.0 (0.2s + 0.25s, SINE)
색 ACCENT_VERMILION 빨강
반복 매 round 후 1회 (round 안에서 호흡하지 않음)

→ 제안 (신규): 임박 영역은 반복 호흡 (0.6s loop, 1.05 ↔ 1.15)? 너무 부담스러우면 1회만.

2-4. pot 카운트업 (Board.gd._animate_pot)

4. UI 트랜지션

4-1. 화면 전환 (scene change)

현재: Godot 기본 (즉시 전환).

제안 — 신규:

단계 모션
1. 페이드 아웃 검은 오버레이 alpha 0 → 1 (0.2s)
2. scene 교체 instant
3. 페이드 인 alpha 1 → 0 (0.3s)
총 0.5s

부드러운 전환 + 로딩 가림.

4-2. 버튼 hover / pressed

상태 모션
hover scale 1.0 → 1.03 (0.1s, SINE_OUT)
pressed scale → 0.97 (0.05s instant) → 1.0 (0.1s)
비활성화 modulate.a 1.0 → 0.5 (instant)

4-3. BettingPanel 모드 전환 (Sprint 3.13)

1차 (주홍) → 2차 (금) 진입 시.

단계 모션
1. 1차 버튼 fade out alpha 1 → 0 (0.15s)
2. 2차 버튼 fade in alpha 0 → 1 (0.15s)
3. ante_label 색 transition color tween (0.2s)
4. 2차 슬라이더 slide-in width 0 → full (0.25s, BACK_OUT)

9. 파일 납품 형식

9-1. After Effects → Lottie

각 모션 별 .json (Lottie) 으로 납품. Godot 측에서 Lottie 라이브러리 또는 자체 Tween 으로 재현.

assets/motion/
├── M-01_card_fade_in.json
├── M-02_card_flip.json
├── ...

9-2. 사양 시트 (스프레드시트 / Notion)

각 모션의 timing 표를 디자이너 ↔ 개발자가 공유. 변경 시 git PR 로 추적.

10. 개발자 측 구현 참고

현재 Godot Tween 코드 위치:

  • 카드 모션: scripts/game/Card.gd
  • 점수 / 명성치 / pot: scripts/game/Board.gd
  • 마패 발동: scripts/game/MapaSlot.gd
  • 화면 전환: 각 화면의 _on_back() 등 — 현재 직접 change_scene_to_file (트랜지션 없음)

제안 모션 (M-04 mulligan 교체, M-09 토스트, M-10 scene change 등) 은 코드 추가 필요. 디자이너 시안 확정 후 별도 sprint 에서 구현.
