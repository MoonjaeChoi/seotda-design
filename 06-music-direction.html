K-Sutda — 음악 방향서

BGM 3개 + SE 8개의 톤·악기·길이·라이선스 가이드. 작곡가 / 사운드 디자이너 / 음원 큐레이터용.

0. 전체 방향

0-1. 한 줄 정의

"한국 전통 악기 (가야금·대금·장구·북) 의 무게감 + 모던 신스 / 미니멀 비트의 긴장감 융합"

0-2. 참고 작품

  • 발라트로 BGM (LouisF) — 미니멀 + 재즈, 카드 게임 톤. 톤 참고만, 모방 ❌
  • Sekiro: Shadows Die Twice OST (Yuka Kitamura) — 동양 전통 + 긴장감 융합
  • Persona 5 Strikers — 재즈 + 일렉트로닉 + 동양 악기
  • 김덕수 사물놀이 — 한국 전통 타악
  • 이날치 + 앰비규어스 댄스 컴퍼니 — 한국 전통 + 모던 융합 톤

0-3. 톤 원칙

  • ✅ 차분 + 무게감 — 한국 도박판의 진중함
  • ✅ 긴장 + 즉각성 — 게임으로서의 몰입
  • ✅ 한국 전통 악기 50% 이상
  • ❌ 카지노 / 라스베가스 톤 (밝은 빅밴드 ❌)
  • ❌ 일본풍 (샤미센·고토 등 — 한국 가야금과 구분)
  • ❌ 록 / 메탈 (게임 톤 안 맞음)

1. BGM 3개

1-1. bgm_menu — 메뉴 음악

사용처: MainMenu, Leaderboard, ProbabilityInfo, HelpInfo, DailyActivity, Market

항목 사양
길이 2~3분 (loop 매끄럽게)
분위기 차분, 약간 신비, 한국 전통
주요 악기 가야금 (메인 멜로디) + 대금 + 신스 패드
BPM 60~80 (느림)
키 단조 (Minor) — 자연스러운 한국 정서
볼륨 -18dB ~ -14dB (대화 톤)

구성 제안:

0:00  가야금 솔로 (4 마디)
0:15  대금 합주 (낮은 옥타브)
0:45  신스 패드 추가 — 깊이감
1:30  가야금 변주
2:30  fade-out → loop 시작

감정 키워드: 정적, 사색, 노름판 한 가운데 들어서기 직전의 침묵.

1-2. bgm_game — 게임 플레이 음악

사용처: Board (단계 1~8, 일반 라운드)

항목 사양
길이 3~5분 loop
분위기 긴장 + 집중, 적당한 추진력
주요 악기 장구 (낮은 비트) + 가야금 (멜로디) + 신스 베이스
BPM 100~120 (보통)
키 단조
볼륨 -20dB (UI / SE 와 균형)

구성 제안:

0:00  장구 4박자 비트 (8 마디 인트로)
0:30  가야금 멜로디 진입
1:00  대금 또는 신스 추가 — 레이어 증가
2:00  비트 다운 → 가야금 솔로 (긴장 완화)
2:30  비트 다시 → 클라이맥스
4:30  fade-out → loop

감정 키워드: 카드 한 장 한 장 신중하게, 베팅 라운드의 압박.

1-3. bgm_boss — 보스 라운드 (선택)

사용처: Board 의 round 3 (최종 라운드, 단계 보스 등장)

항목 사양
길이 2~3분 loop
분위기 강한 긴장, 결단의 순간
주요 악기 사물놀이 (장구·꽹과리·징·북 모두) + 신스
BPM 130~150 (빠름)
키 단조, 가끔 long crescendo
볼륨 -16dB (좀 더 크게)

구성 제안:

0:00  꽹과리 강렬한 인트로 (4 마디)
0:15  장구 + 북 풀 비트
0:45  가야금 + 대금 — 멜로디 레이어
1:30  텐션 빌드 (crescendo)
2:00  클라이맥스 — 결단의 순간
2:30  loop

감정 키워드: 운명의 판, 결투, 마지막 한 판.

→ 베타 직전에는 bgm_game 하나만으로 시작 가능. bgm_boss 는 베타 후 추가.

2. SE (효과음) 8개

2-1. 카드 관련 (3개)

파일 길이 톤
card_flip.ogg 0.2~0.3s 종이 / 나무 카드 뒤집는 소리. 살짝 마찰음.
card_deal.ogg 0.3~0.4s 카드 던지거나 미끄러지는 소리. 빠른 연속도 가능.
(선택) card_shuffle.ogg 1~2s 카드 섞기 — 라운드 시작 시. 현재 미사용.

음원 톤: 진짜 화투 (두꺼운 카드, 약간 마찰) 또는 한국 전통 패 톤.

2-2. 결과 관련 (2개)

파일 길이 톤
win.ogg 0.8~1.2s 짧은 한국 전통 fanfare. 가야금 또는 박수 1회.
lose.ogg 0.6~0.8s 어두운 한숨 / 종소리 / 가야금 단음 하행.

감정: win 은 미소 짓는 정도, 점프 임팩트 ❌. lose 는 한숨 / 안타까움, 비극 ❌.

2-3. UI / 상호작용 (1개)

파일 길이 톤
button_click.ogg 0.1s 매우 짧고 깔끔. 나무 막대 두드림 또는 한국 박 (拍子) 톤.

2-4. 게임 메커니즘 강조 (3개)

파일 길이 톤 발동
mapa_trigger.ogg 0.5~0.7s 신비로운 종소리 / 가야금 코드 1회 / 풍경 (風磬) 마패 발동
first_dia.ogg (Sprint 3.13 신규) 0.4s 작은 한숨 + 카드 떨어지는 소리. 단호함보다 살짝 아쉬움. AI 1차 dia
bluff_success.ogg (Sprint 3.13 신규) 0.6s 가면 벗는 / 키득거림 / 단소 짧은 트릴 — 약간 장난기 블러프 성공

3. 사양 / 포맷

3-1. 파일 포맷

종류 포맷 비트레이트 채널 비고
SE .ogg Vorbis 64~128 kbps mono 가벼움 우선
BGM .ogg Vorbis 128~192 kbps stereo loop 메타 포함

3-2. Loop 처리

BGM 은 매끄러운 loop:

  • Ogg Vorbis 의 LOOPSTART / LOOPEND 메타 태그 사용
  • 또는 wav 원본 + Audacity 로 fade out → fade in 겹쳐 자연스럽게
  • 마지막 박자가 첫 박자로 자연스럽게 이어지도록

3-3. 볼륨 균형

종류 권장 dBFS
BGM (메뉴) -18 dB
BGM (게임) -20 dB
BGM (보스) -16 dB
SE 일반 -12 dB
SE 강조 (win/mapa) -10 dB
SE UI (button) -16 dB

→ 사용자가 옵션으로 BGM/SE 별도 슬라이더로 조절 가능해야 (별도 sprint).

3-4. 파일 명명 규칙

assets/audio/
├── bgm_menu.ogg
├── bgm_game.ogg
├── bgm_boss.ogg
├── card_flip.ogg
├── card_deal.ogg
├── win.ogg
├── lose.ogg
├── button_click.ogg
├── mapa_trigger.ogg
├── first_dia.ogg

→ scripts/core/AudioBus.gd 의 SOUNDS dict 키와 일치.

4. 라이선스

4-1. 옵션

옵션 장점 단점
작곡가 외주 (work-for-hire) 톤 완벽 일치, unique 비용 + 시간
CC0 / royalty-free 무료 / 즉시 톤 일치 어려움, 다른 게임에서도 사용됨
Audiio / Epidemic Sound (구독) 다양 + 합법 월 구독 비용, 게임 영구 사용 라이선스 확인 필요
본인 작곡 무료, 톤 완벽 작곡 가능자에 한해

4-2. 추천 — 단계적

  1. 베타: CC0 / royalty-free 임시 음원으로 시작 (Freesound, OpenGameArt 등)
  2. 베타 피드백 후 정식 배포: 작곡가 외주로 BGM 3개 + 핵심 SE 5~8개 교체

4-3. 한국 전통 악기 음원 소스

  • 국립국악원 디지털 음원 — 일부 공개, 라이선스 확인
  • 한국저작권위원회 공유저작물 — 한국 전통 곡 일부 public
  • Wonderful Things — 한국적 royalty-free (가야금·대금 일부)

4-4. 라이선스 동봉

  • 모든 음원 출처 + 라이선스 텍스트를 LICENSES.md 에 명시
  • 게임 내 크레딧 화면 (옵션) 에 작곡가 / 음원 출처 노출

5. 음향 톤 가이드 (작곡가 / 사운드 디자이너용)

5-1. Do ✅

  • 가야금 / 대금 / 장구 / 북 / 꽹과리 (한국 전통 5종)
  • 신스 패드 (Ambient pad) — 차분한 분위기 보강
  • 동양적 5음 음계 (pentatonic minor) — 자연스러운 한국 정서
  • 미니멀 — 음원이 너무 빽빽하면 UI 사운드 묻힘

5-2. Don't ❌

  • 일본 악기 (샤미센 / 고토) — 한국 가야금과 시각·청각 구분 어려움
  • 빅밴드 / 라스베가스 카지노 톤
  • 메탈 기타 / 록 드럼
  • 보컬 (가사) — 게임 집중 방해
  • 갑작스러운 큰 변동 (다이내믹 레인지 너무 크면 사용자 놀람)

5-3. 감정 매핑

게임 상황 음악 / SE 톤
메인 메뉴 진입 정적, 사색 (bgm_menu 시작)
새 런 시작 bgm_menu fade-out → bgm_game crossfade
1차 베팅 bgm_game 유지
2차 베팅 (긴장 ↑) bgm_game 그대로 (가볍게 텐션 빌드 OK)
결과 — 승리 win.ogg + bgm_game 살짝 lift
결과 — 패배 lose.ogg + bgm_game 살짝 sink (옵션)
단계 클리어 짧은 fanfare → Market
보스 라운드 진입 bgm_game → bgm_boss crossfade (1초)
게임오버 모든 음악 fade-out → 침묵 1초 → MainMenu
런 클리어 bgm_boss → 짧은 win → fade-out → MainMenu

7. 통합 / 테스트 (개발자용)

7-1. 통합

음원 파일을 assets/audio/ 에 배치만 하면 자동 인식 (AudioBus.gd 코드 변경 없음).

7-2. 인게임 테스트

  • 헤드리스 테스트는 사운드 검증 안 함 → 에디터에서 직접 확인
  • 볼륨 균형: 메뉴 → 게임 → 결과 흐름 한 번 통째로 들어보기
  • 사양 정확: card_flip 이 너무 길면 빠른 베팅 시 겹침 (≤ 0.3s 유지)

7-3. 옵션 (향후 sprint)

  • 사용자 옵션 화면 신규 — BGM / SE 볼륨 슬라이더 (0~100)
  • 음악 끄기 옵션 (mute toggle)
  • 접근성: 자막 (효과음 → 텍스트 표시)
