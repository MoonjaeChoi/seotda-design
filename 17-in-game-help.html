K-Sutda — 인앱 도움말 / 튜토리얼 가이드

HelpInfo 화면 강화 + 첫 런 인터랙티브 튜토리얼 + 컨텍스트 hint. 사용자 (특히 P5 캐주얼) 가 5분 안에 게임을 이해하도록.

0. 도움말 시스템 전략

0-1. 3-layer 도움말 구조

Layer 발동 깊이 대상
Layer 1 인터랙티브 튜토리얼 첫 런 자동 핵심만 P5 캐주얼 (필수)
Layer 2 컨텍스트 hint 특정 상황 첫 발생 보통 P2 / P4 / P5
Layer 3 HelpInfo 정적 화면 사용자 명시 요청 모든 정보 P1 / P3 / P6 (학습 의지)

각 Layer 는 선택적 — 스킵 가능 + 다시 보기 가능.

0-2. 디자인 원칙

  • 방해 최소 — 튜토리얼 강제 X, 스킵 가능
  • 인터랙티브 — 읽기보다 실제 행동으로 학습
  • 점진 노출 — 첫 런에 모든 것 보여주지 X, 단계별 새 기능 만났을 때
  • 사용자 페이스 — 시간 제한 없음 (P2 / P5 천천히)

1. Layer 1 — 첫 런 인터랙티브 튜토리얼

1-1. 발동 조건

  • 신규 사용자 (Steam Cloud / 로컬 첫 실행)

  • 또는 MainMenu → "📖 게임 규칙" → "튜토리얼 다시" 버튼

1-3. 튜토리얼 UI 컴포넌트

  • Tutorial Overlay: 검은 alpha 0.6 overlay + 특정 영역 spotlight (cut-out)
  • Tutorial Toast: 화면 중앙 / 하단 카드형 (24px Bold + 본문)
  • Skip Button: 우상단 작은 "튜토리얼 건너뛰기"
  • Highlight Pulse: 활성 영역 (콜 버튼 등) 펄스 (1초 loop)

1-4. 튜토리얼 시드 (디자인 결정)

  • 고정 시드 추천: 첫 런 튜토리얼은 결정성 보장 시드 (예: seed=0)

    • 첫 카드 = 1월 송학광 (광패)
    • 두 번째 카드 = 1월 매조 (1땡)
    • AI = 약한 패 (콜 하면 승리)
    • → 사용자 첫 경험 = 승리 (긍정적)

  • 두 번째 런부터: 무작위 시드 (정상 게임)

2. Layer 2 — 컨텍스트 Hint

2-1. 발동 시점

특정 상황 첫 발생 시 한 번만 짧은 안내 (1.5~2초 토스트):

상황 Hint
mulligan 첫 가능 (두 번째 카드 받은 직후) "💡 카드를 클릭하면 8명성치로 재배분 가능"
1차 dia 첫 손실 "💡 1차 다이는 -5만 손실. 약한 첫 카드면 일찍 빠지세요"
AI 1차 dia 첫 발생 "💡 AI 가 약하다고 판단해 빠졌습니다. 작은 보상 +5"
블러프 첫 성공 "🃏 블러프 성공! 약한 패 + 레이즈 + AI 다이 = 시스템 발동"
광땡 첫 완성 "🌟 광땡! K-Sutda 최강 족보 — 점수 ×10"
명성치 위험 영역 첫 진입 "⚠ 명성치 < 0. 신중한 베팅 필요"
명성치 임박 영역 "⚠️ 게임오버 -50 명성치 남음"
단계 5 첫 도달 "🎯 5단계 — DEFENSIVE AI 등장. 신중하게 빠지는 상대"
단계 7 첫 도달 "🎭 7단계 — 블러프 마스터 탈춤 영감. 망통으로도 raise 합니다"
단계 8 첫 도달 "👑 8단계 — 전설의 마패 남궁천. 마지막 한 판"

2-2. Hint 디자인

┌──────────────────────────────────────┐
│ 💡 카드를 클릭하면 8명성치로          │
│    재배분 가능                        │
└──────────────────────────────────────┘
화면 상단 중앙. 1.5~2초 후 자동 사라짐.
배경: 한지 베이지 + alpha 0.85
글자: 짙은 먹 16px

2-3. Hint 빈도 제한

  • 같은 hint 는 1회만 표시 (Save 에 기록)
  • "Hint 끄기" 옵션 (Settings — 미래)
  • 매 런 새로 표시할지 / 영구 표시 X 결정 — 영구 (Save) 권장

3. Layer 3 — HelpInfo 정적 화면 (현재 구현)

3-1. 현재 (Sprint 3.15)

scripts/ui/HelpInfo.gd — RichText 정적 안내. 다음 섹션:

  • 2단계 베팅 흐름
  • mulligan
  • 블러프
  • 명성치 한도
  • 점수 공식
  • 단계 진행

3-2. 강화 권장 (베타 후)

3-2-1. 시각 / 일러스트 추가

[현재 텍스트만]
"1차: 카드 1장 받기"

[강화]
[일러스트: 카드 1장 face_up + 카드 1장 face_down 슬롯]
"1차: 카드 1장 받기 — 두 번째 카드는 가려져 있음"

3-2-2. 인터랙티브 데모 (옵션)

"콜 / 1차 레이즈 / 다이 의 차이를 직접 시도해 보세요"
[작은 mini-game 화면 — 가상 카드 + 버튼 3개]
[클릭 → 결과 미리보기]

3-2-3. 섹션 분할 (현재 한 페이지 → 탭)

HelpInfo (TabContainer):
├── [기본] — 2단계 베팅, 명성치, 단계
├── [고급] — 블러프, 마패 시너지, mulligan 전략
├── [캐릭터] — 8 보스 소개 (가벼운)
└── [용어집] — 광/땡/끗/망통 등

3-3. HelpInfo 도달 경로

  • MainMenu → 📖 게임 규칙 (현재)
  • Board → 우상단 ? 버튼 (미래)
  • Settings → 도움말 (미래)
  • 인앱 검색 (옵션, 미래)

4. 일러스트 / 비주얼 요소

4-1. 튜토리얼 일러스트

각 단계 / Hint 옆에 작은 일러스트 (32×32 ~ 80×80):

컨텍스트 일러스트
화투 카드 인지 카드 1장 + 손가락 가리킴
1차 베팅 카드 1장 + 베팅 칩
2차 베팅 카드 2장 + 큰 베팅
결과 트로피 / 가면 (승/패)
Mulligan 카드 ↔ 카드 (교체)
블러프 성공 가면
광땡 두 광패 + 황금 글로우
위험 경고 표시

4-2. HelpInfo 본문 일러스트

긴 텍스트 사이사이 일러스트 (300×200 정도):

  • 2단계 베팅 흐름도 (도식)
  • 점수 공식 (수식 시각화)
  • 단계 진행 (1~8 진행 도식)
  • 캐릭터 보스 8명 thumbnails

→ 디자이너 작업 분량 추정: 일러스트 15~20장 (튜토리얼 + HelpInfo)

5. 카피 / 톤 — 도움말 어휘

5-1. 톤

  • 친근 + 격조 — "안녕하세요!" + "...좋은 한 판 되시길!"
  • 명령형 적극 — "지금 [콜] 을 눌러 보세요"
  • 한국적 — "광 한 번 보세요" / "다이로 빠지세요"

5-2. 핵심 어휘 일관 (10-localization §2 참조)

도움말 어휘
베팅 액션 콜 / 레이즈 / 올인 / 다이 (1차) / 폴드 (2차)
손실 표현 "명성치 -5" (정확) > "5 잃기" (모호)
승리 표현 "패 점수 × 마패 배수 = final score" > "이긴다!"
위험 표현 "위험 영역 / 게임오버 임박" > "조심!"

5-3. 한↔영 카피 예시

[KO]
"💡 카드를 클릭하면 8명성치로 재배분 가능"

[EN]
"💡 Click a card to redraw (costs 8 prestige)"

→ 10-localization 의 i18n 키에 도움말 카피도 포함.

7. 접근성 (도움말)

(09-accessibility 와 일관)

  • 모든 도움말 텍스트 14px 이상
  • 명도 대비 4.5:1
  • 사운드 효과에 시각 대안 (도움말도 시각 위주)
  • 키보드 네비 (Tab + Enter)
  • Reduced Motion 시 spotlight / 펄스 즉시 (모션 X)

8. 도움말 작업 우선순위

Tier 1 — 베타 직전

  • HelpInfo 화면 (Sprint 3.15 — 정적 RichText 완성)
  • 컨텍스트 hint 5종 (필수: mulligan, 블러프, 위험 영역, 광땡, 단계 8)
  • hint 발동 기록 (SaveManager)

Tier 2 — 베타 후

  • 첫 런 인터랙티브 튜토리얼 (T1~T5)
  • 튜토리얼 일러스트 (15~20장)
  • HelpInfo 섹션 분할 (탭)
  • HelpInfo 일러스트 추가

Tier 3 — 폴리시

  • 인터랙티브 데모 (HelpInfo 안)
  • 인앱 검색 (도움말 키워드)
  • 비디오 가이드 (옵션, 영상 작가)

9. 도움말 텍스트 단어 수 가이드

종류 단어 수
튜토리얼 단계 본문 3060자 (12 문장)
컨텍스트 hint 20자 이내
HelpInfo 섹션 200~400자 (1 화면)
Hint 사운드 0 (시각만)

→ 짧고 명확. 긴 설명 ❌.

10. 디자이너 ↔ 카피라이터 협업

10-1. 카피라이터 작업 분량

  • 튜토리얼 5단계 × 3060자 = 150300자
  • Hint 10개 × 20자 = 200자
  • HelpInfo 강화 본문 600~1200자

→ 총 1000~1700자 정도 (한국어 + 영어 양쪽)

10-2. 디자이너 작업 분량

  • 튜토리얼 일러스트 15~20장
  • HelpInfo 일러스트 8~10장
  • Tutorial Overlay / Spotlight 디자인 (1 컴포넌트)

10-3. 검증

  • P5 (캐주얼) 사용자 베타 — 튜토리얼 후 5분 안에 첫 라운드 완료?
  • P2 (화투 친숙) — 추가 정보 없이 즐길 수 있나?
  • P3 (글로벌) — 영문 자연스러운가?

11. 결론

좋은 도움말 = 사용자가 도움말이 있다는 사실을 잊고 게임에 몰입.

3-layer 구조:
1. 첫 만남 — 인터랙티브 튜토리얼 (5분)
2. 상황별 — 컨텍스트 hint (한 번씩 짧게)
3. 학습 의지 — HelpInfo 화면 (언제든)

목표: P5 (캐주얼) 가 5분 안에 첫 라운드 완료.
