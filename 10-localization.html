3. 톤 차이 (한↔영)

3-1. 한국어 톤

  • 간결, 결단적, 한국적 — "다이", "한 판 더", "광 보임"
  • 존댓말 X (게임 내 대사는 평어 — 도박판 분위기)
  • 한자 / 한국어 혼용 OK ("명성치 (名聲)" — 무게감)

3-2. 영어 톤

  • 결단적, 약간 archaic / mythic — "Fold", "One last hand", "The legend"
  • 도박 슬랭 사용 OK ("All-In", "Bluff", "Showdown") — 카드 게임 자연스러움
  • "Game Over" 같은 일반 게임 표현 OK
  • 한국 문화 단어 유지: "Sutda", "Mapa", "Gwangddaeng" (영어로 표시 + 첫 등장 시 설명)

3-3. 한국 고유 명사 영문 표기

한국어 권장 영문 표기 이유
섯다 Sutda 보편 사용
마패 Mapa 영화 "마패" 글로벌 인식
광 (光) Gwang 한자 발음
광땡 Gwangddaeng 보존 (괄호 설명: "highest hand")
장땡 Jangddaeng 보존
알리 / 독사 / 구삥 / 장삥 Ali / Doksa / Gupping / Jangpping 보존
끗 (점수) Keut 보존 (괄호 설명)
망통 Mangtong 보존 ("zero hand")
명성치 Prestige 의역 (가장 가까운 영어)

→ 한국 고유 명사 유지로 게임 정체성 보존. 영문 사용자도 "Mapa" "Gwangddaeng" 검색 가능.

4. 글자 길이 변화 대응

4-1. 한↔영 길이 비율

일반적으로 영문 = 한글 × 1.3 ~ 1.5.

예시:

한국어 (자) 영문 (자) 비율
다이 (2) Fold (4) 2.0
새 런 시작 (5) New Run (7) 1.4
명성치: 1500 (7) Prestige: 1500 (13) 1.9
카드 재배분 가능 (8) Mulligan available (17) 2.1

4-2. 디자인 대응

  • 버튼 폭 유동적: min-width 설정, 텍스트 따라 확장
  • 라벨 wrap 허용: 한 줄 강제 X (또는 ellipsis "...")
  • 카드 영역 고정: 텍스트 길이 변동 무관하도록 (카드 자체 220×320 px 고정)
  • 테스트: 가장 긴 언어 (영문 / 독일어 — 미래) 기준 레이아웃 검증

6. i18n 구현 (개발자 기술 가이드)

6-1. Godot 4 — TranslationServer

# 1. .csv 파일에 키-값 저장
#   keys, ko, en
#   menu.start, "새 런 시작", "New Run"
#   betting.first.dia, "다이 (-%d)", "Fold (-%d)"

# 2. Project Settings → Localization → Translations → 추가

# 3. 코드에서:
print(tr("menu.start"))  # 현재 언어로 자동 출력

# 4. 동적 인자:
print(tr("betting.first.dia") % first_ante)

6-2. 파일 위치

assets/

6-3. 언어 전환 옵션

Settings → Language:
  ◉ 한국어 (Korean)
  ○ English

  ○ 中文 (Chinese)

→ 현재 코드에 옵션 화면 없음 — Phase 4 별도 sprint.

6-4. 디자이너 작업 형식

번역가 ↔ 디자이너 ↔ 개발자 협업:

  • 번역가: CSV 파일 작성 (위 표 기반)
  • 디자이너: 가장 긴 언어 기준 레이아웃 검증
  • 개발자: tr() 함수로 통합 + 폰트 fallback 적용

8. 컬처럴 어댑테이션 (Cultural Adaptation)

8-1. 한국 문화 어휘 — 영문 어떻게 전달

한국 문화 영문 전략
명성치 "Prestige" (정확한 영어 대응 — 사회적 명예)
광패 "Gwang" (한국 고유) + "(Brightness/Star Cards)" 첫 등장 시
마패 "Mapa" + "(Gambler / Master)" 첫 등장 시 — 영화 "마패" 글로벌 인지
단계 보스 이름 한국 이름 그대로 (Dalsamee, Yeongja, Dojin, Gangsu, Hyunmuk, Chaerin, Talchum, Nam-gung Cheon)

8-2. 도박 톤 문화 차이

한국 영어권
"다이" (조용히, 무게감) "Fold" (포커 톤, 신속)
"광 한 번" (인생의 한 번) "One last hand" (운명)
"운이 좋군" (체념·인정) "Lucky" (단순 운)

→ 직역하지 말고 톤 살리기.

8-3. 사행성 모사 회피 — 영문도 동일

  • "Win money" / "Earn cash" ❌
  • "Score points" / "Build prestige" ✅
  • 모든 disclaimer 영문 번역 시 같은 무게감

9. QA 체크리스트

9-1. 디자이너 QA

  • 가장 긴 텍스트 기준 레이아웃 깨짐 없음
  • 한글 / 영문 fallback 폰트 시각 일관성
  • 폰트 사이즈 14px 이상 (영문 12px ~ 한글 14px 차이 고려)

9-2. 번역가 QA

  • 모든 키 번역 완료 (빈 값 없음)
  • 동적 인자 (%d, %s) 모두 보존
  • 한국 고유 명사 일관성 (Sutda / Mapa / Gwang)
  • 톤 일관성 (결단적 / 한국적 정서 유지)

9-3. 개발자 QA

  • tr() 함수로 모든 텍스트 wrapping
  • 한↔영 전환 시 즉시 갱신 (재시작 불필요)
  • CSV 인코딩 UTF-8 BOM
