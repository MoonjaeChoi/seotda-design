K-Sutda — 분석 / KPI / 메트릭 가이드

KPI 정의 / 추적 이벤트 / 대시보드 디자인 / 베타 vs 정식 출시 측정. 사용자 행동 / 게임 디자인 / 마케팅 성과.

0. 분석 전략

0-1. 왜 분석인가

  • 솔로 개발자 → 직관 ≠ 사용자 행동. 데이터로 검증 필수
  • 베타 → 정식 출시 → 베타 데이터 = 정식 출시 결정 기반
  • 밸런스 / 디자인 → 데이터 기반 개선 (감 ❌)
  • 마케팅 → 어떤 채널이 효과적인지 측정

0-2. 추적 원칙

  • ✅ 사용자 동의 후 익명 데이터만
  • ✅ 개인정보 (이메일 / 실명) 추적 ❌
  • ✅ 게임 행동 (세션 / 단계 / 베팅) 위주
  • ❌ 광고 / 마케팅 추적 (페이스북 픽셀 등) ❌ — 인디 게임 톤
  • ✅ 한국 개인정보보호법 / GDPR 준수

1. 핵심 KPI (Key Performance Indicators)

1-1. 사용자 KPI

KPI 정의 목표 (베타 / 정식)
DAU (Daily Active Users) 하루 게임 실행자 베타 30 / 정식 500+
WAU (Weekly Active) 1주일 게임 실행자 베타 80 / 정식 1,500+
MAU (Monthly Active) 1개월 게임 실행자 베타 100 / 정식 3,000+
DAU/MAU 스티키니스 (얼마나 자주 돌아오나) 0.2 (보통) → 0.4 (좋음)
신규 가입 신규 사용자 수 베타 일 5명 / 정식 일 30명
재방문율 (Retention) Day 1 / 7 / 30 재플레이 D1: 40% / D7: 20% / D30: 10%

1-2. 게임플레이 KPI

KPI 정의 목표
세션 길이 평균 1회 플레이 시간 20~30분
세션 / 사용자 1주일 평균 세션 수 3~5회
런 완주율 단계 1 시작 → 단계 8 클리어 5~15% (어려운 게임)
단계별 통과율 각 단계 통과 % 1: 80%+ / 8: 5~10%
평균 단계 도달 run 당 도달 단계 평균 3~4 단계
마패 카드 사용 가장 자주 장착된 카드 TOP 10 매주 측정
블러프 시도율 약한 패로 raise 한 라운드 % 5~15%
mulligan 사용율 두 번째 카드 받은 후 재배분 % 20~30%
게임오버 / 명성치 -150 한도 도달 % 5~10%

1-3. 수익 KPI (정식 출시 후)

KPI 정의
첫 주 판매 출시 첫 주 판매 수
누적 판매 전체 판매
환불율 Steam 환불 % (< 5% 목표)
DLC 전환 OST / Art Pack DLC 구매 %
굿즈 판매 굿즈 매출 (보조)
ARPU (Average Revenue per User) 사용자당 평균 매출

1-4. 마케팅 KPI

KPI 정의
Steam 페이지 조회 일별
위시리스트 가입자 수 / 가입율
트레일러 조회 YouTube / Twitter
인플루언서 키 사용율 영상 / 스트림 1회 이상 %
SNS 팔로워 Twitter / Discord
리뷰 점수 Steam / App Store / Google Play

2. 추적 이벤트 (Event Tracking)

2-1. 게임 진행 이벤트

// 추적할 이벤트 (예시 — 실제 구현은 Godot + 자체 백엔드 또는 Mixpanel 등)

// 세션
session_start { user_id, platform, version }
session_end { user_id, duration_s }

// 런
run_start { seed, starting_mapa[3] }
run_end {
  status: "victory" | "defeated" | "abandoned",
  final_prestige: int,
  stage_reached: int,
  duration_s: int,
  rounds_played: int
}

// 단계
stage_start { stage: int }
stage_clear { stage: int, prestige_at_end: int }

// 라운드 (집계 — 너무 많으면 부담)
round_summary {
  stage: int,
  result: "win" | "lose" | "draw",
  ending: "showdown" | "fold" | "first_dia",
  player_bluffed: bool,
  player_first_action: "call" | "raise" | "dia",
  mulligan_used: bool
}

// 시장
market_visited { stage: int }
mapa_purchased { mapa_id, rarity, price }
mapa_swapped { from_slot, to_slot }
soul_card_discovered { mapa_id }  // Legendary 5% 확률

// UI
help_opened { source: "main_menu" | "in_game" }
daily_activity_viewed
leaderboard_viewed

// 위험 영역
prestige_danger_entered { stage: int }
prestige_imminent_entered { stage: int }
gameover_triggered { stage: int, final_prestige: int }

2-2. 사용자 행동 이벤트

nickname_set { length }
language_changed { from, to }
audio_settings_changed { bgm_volume, se_volume }
reduced_motion_enabled
help_dismissed { tutorial_step }

2-3. 에러 / 크래시 이벤트

crash { type, stack_trace_hash, platform, version }
error_save_load { reason }
network_error { endpoint, error_code }

3. 데이터 수집 방식

3-1. 옵션 1 — Supabase (현재 사용 중)

[장점]
- 이미 leaderboard 에 사용 — 인프라 확장 쉬움
- PostgreSQL — SQL 쿼리 가능
- 무료 tier 충분 (베타 100명)

[단점]
- 분석 대시보드 별도 구축 필요
- 시각화 도구 (Metabase / Grafana) 별도

[구현]
events 테이블:
  id, user_id (anon), event_type, timestamp, payload (JSON), platform

3-2. 옵션 2 — Mixpanel / PostHog

[장점]
- 분석 전용 SaaS — 대시보드 강력
- 코호트 / 퍼널 / 리텐션 분석 자동

[단점]
- 무료 tier 제한 (월 X 이벤트)
- 외부 서비스 의존

[구현]
HTTP API 호출 (Godot HTTPRequest)

3-3. 옵션 3 — Steam 자체 분석

[장점]
- Steam 통합 (자동 추적 일부)
- DAU / 세션 / 환불 / 리뷰 자동

[단점]
- 자세한 게임 내 이벤트 추적 X
- 모바일 / 다른 플랫폼 X

[사용]
Steam Workshop / Stats API

3-4. 추천 — Hybrid

  • Steam (PC 자동) — DAU / 세션 / 판매
  • Supabase (자체) — 게임 내 이벤트 / 통계
  • Mixpanel (옵션) — 퍼널 / 코호트 (예산 여유 시)

11-2. 데이터 사용 정책

[K-Sutda 데이터 수집 정책]

수집 항목:
- 익명 게임 행동 (세션 / 단계 / 베팅)
- 디바이스 정보 (OS / 화면 크기 — 기술 지원)

수집 안 함:
- 이메일 / 실명 / 위치
- 광고 추적
- 외부 서비스 공유

사용 목적:
- 게임 개선 (밸런스 / 버그)
- 통계 (집계 데이터만)

권리:
- 언제든 수집 거부 (옵션 메뉴)
- 데이터 삭제 요청 (이메일 문의)

12. 작업 우선순위

Tier 1 — 베타 직전

  • Steam 자동 분석 (DAU / 세션)
  • Supabase events 테이블 생성
  • Godot → Supabase 이벤트 전송 (session_start / run_end)
  • 데이터 수집 동의 UI

Tier 2 — 베타 운영 중

  • Metabase 대시보드 셋업 (메인 KPI)
  • 단계별 통과율 차트 자동
  • 코호트 분석 (재방문)
  • 주간 리포트 자동 (개발자 메일)

Tier 3 — 정식 출시 후
