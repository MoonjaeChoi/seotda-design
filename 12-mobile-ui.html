K-Sutda — 모바일 UI 적응 가이드

데스크탑 (가로 1280×720+) 위주의 현재 UI 를 모바일 (세로 위주) 로 재배치. iOS / Android 양쪽 + 안전 영역 + 터치 타겟.

0. 모바일 출시 전략

0-1. 우선순위

플랫폼 출시 시점 비고
iOS (iPhone) 🟡 Phase 4 6.5"~6.7" 메인 (iPhone 11 Pro Max ~ 15 Pro Max)
iPad 🟢 Phase 4 후 가로/세로 모두 지원 (iPad 12.9")
Android (Phone) 🟡 Phase 4 일반 1080×1920 ~ 1080×2400
Android Tablet 🟢 Phase 5 별도 적응 (iPad 와 유사)

0-2. 화면 방향 결정

옵션 장점 단점
세로 (Portrait) 전용 한 손 플레이, 모바일 게임 표준 데스크탑 UI 큰 재배치 필요
가로 (Landscape) 전용 데스크탑 UI 그대로 활용 한 손 플레이 불가, 모바일 자연스럽지 X
자동 회전 지원 사용자 선택 양쪽 레이아웃 모두 디자인·테스트

🎯 추천: 세로 (Portrait) 전용

  • 한국 모바일 게임 사용자 한 손 플레이 선호
  • 카드 게임 특성상 세로 배치 자연스러움 (Hearthstone 모바일 등 참고)
  • iPad 는 자동 회전 추가 고려 (선택)

1. 화면 해상도 / 안전 영역

1-1. 기준 해상도 (디자인 작업용)

디바이스 논리 해상도 물리 해상도 비고
iPhone 13 Pro 390 × 844 1170 × 2532 작은 디자인 기준
iPhone 15 Pro Max 430 × 932 1290 × 2796 큰 디자인 기준
Android (1080×2400) 360 × 800 1080 × 2400 일반 안드로이드
Android (1080×1920) 360 × 640 1080 × 1920 구형 안드로이드

디자인 기준: 390 × 844 (iPhone 13 Pro) 를 기본 캔버스로 작업 → 다른 사이즈로 자동 스케일.

1-2. 안전 영역 (Safe Area)

┌─────────────────────┐
│ 상단 노치/Dynamic   │ 47~59px (iOS)
│ Island / 상태바     │ 24px (Android)
├─────────────────────┤  ← Safe Area 시작
│                     │
│   콘텐츠 영역       │  ← 모든 UI 가 여기에
│                     │
├─────────────────────┤  ← Safe Area 끝
│ 홈 인디케이터       │ 34px (iOS)
│ 또는 시스템 바      │ 48px (Android)
└─────────────────────┘

규칙:

  • 모든 인터랙티브 요소는 안전 영역 안에
  • 배경 / 장식만 안전 영역 밖으로 확장 가능
  • iOS / Android 양쪽 자동 처리 (Godot SafeAreaContainer)

3. 터치 인터랙션

3-1. 최소 터치 타겟

디자인 가이드 최소
iOS HIG 44 × 44 pt
Material Design 48 × 48 dp
K-Sutda 권장 48 × 48 pt (양쪽 만족)

→ 모든 버튼 / 클릭 가능 영역 ≥ 48×48.

3-2. 터치 vs 마우스 차이

차이 마우스 (데스크탑) 터치 (모바일)
Hover 있음 (사전 시각 피드백) 없음
클릭 정확도 픽셀 단위 손가락 (큰 영역)
우클릭 가능 없음 (long press 대체)
Tooltip hover 시 표시 (없음 — long press 또는 별도 UI)

대응:

  • Hover 상태 시각 → Pressed 상태 강화로 대체 (터치 후 즉시 피드백)
  • Tooltip → 카드 자체에 정보 일부 노출 (Sprint 3.8 partial reveal 처럼)
  • 우클릭 (카드 상세) → long press (0.5초) 로 매핑

3-3. 스와이프 제스처 (선택 — 미래)

제스처 액션
Swipe Up (카드) 카드 사용 / 베팅 콜 (옵션)
Swipe Down (카드) 카드 다이 / mulligan (옵션)
Pinch Zoom (카드) 카드 자세히 보기 (옵션)

→ Phase 4 폴리시 sprint. 베타에선 버튼만으로 충분.

4. 폰트 사이즈 — 모바일

종류 데스크탑 모바일 권장
TITLE 96px 72px (또는 64px)
HEADING 32px 28px
SUBHEAD 24px 22px
BODY 16px 16px (최소 유지)
CAPTION 12px 13px (작은 화면 가독성 ↑)
SCORE 96px 64px (점수 폭발)

→ 폰트 사이즈 살짝 줄이지만 BODY/CAPTION 은 최소 유지.

5. iOS vs Android 차이

5-1. 디자인 차이

항목 iOS Android
시스템 폰트 SF Pro / Apple SD Gothic Neo Roboto / Noto Sans KR
안전 영역 Notch / Dynamic Island 상태바
시스템 바 홈 인디케이터 (하단) 내비 바 (하단, 작거나 제스처)
버튼 톤 iOS 곡선적 Material 직선적

5-2. 우리 접근

  • 공통 디자인 — Pretendard 폰트 + 우리 Palette
  • 시스템 폰트 fallback 만 OS 별 (Apple SD Gothic Neo vs Noto Sans KR)
  • 시스템 UI (스테이터스바) 색상은 OS 자동 (Godot 설정)

5-3. 빌드 차이

  • iOS: Apple Developer Account ($99/년)
  • Android: Google Play Console ($25 일회)
  • 빌드: Godot Export → 각 플랫폼별 패키지

6. 성능 / 메모리 (모바일)

6-1. 최적화 포인트

항목 가이드
카드 이미지 @2x (440×640) 최대, @3x 불필요
마패 카드 @2x (400×560)
캐릭터 일러스트 @2x (1200×1800) — 큰 화면만
BGM 192kbps 까지 (저전력 디바이스)
메모리 200MB 미만 권장

6-2. Godot 모바일 최적화

  • 텍스처 압축: ETC2 (Android) / ASTC (iOS)
  • 오디오: Ogg Vorbis (둘 다 지원)
  • 화면 갱신: 60 FPS 기준, 30 FPS fallback (저전력)
  • 절전 모드: 비활성 60초 후 30 FPS 강제

  • 베타에선 로컬만
  • 정식 출시 시 옵션 (Supabase 통합 가능)

8. 가로 모드 / iPad

8-1. iPad — 자동 회전

가로 모드 (1024 × 768):
  데스크탑 UI 거의 그대로 + 약간 큰 텍스트
  
세로 모드 (768 × 1024):
  iPhone 모바일 레이아웃 확대

8-2. 가로 ↔ 세로 전환

  • 회전 시 UI 자동 재배치 (반응형 레이아웃)
  • 게임 진행 중 회전 시: pause + scene transition (옵션)

→ Phase 4 별도 작업.

9. 작업 우선순위

Tier 1 — Phase 4 모바일 출시 핵심

  • Board 화면 세로 레이아웃 시안
  • MainMenu 모바일 시안
  • BettingPanel 모바일 (버튼 사이즈 ↑)
  • Card 시각 모바일 검증 (140×200 가독성)
  • 터치 타겟 48×48 검증

Tier 2 — 베타 후

  • Market 2×2 그리드 시안
  • DailyActivity 세로 stack 시안
  • Leaderboard / HelpInfo 스크롤 검증
  • 햅틱 피드백 매핑
  • iPad 가로 모드 시안

Tier 3 — 폴리시

  • 스와이프 제스처
  • Push Notification
  • 클라우드 세이브
