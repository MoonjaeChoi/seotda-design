K-Sutda — 데이터 시각화 가이드

DailyActivity 차트 + 향후 통계 화면 (마패 사용 빈도 / 단계별 통과율 / 명성치 분포 등) 의 시각 디자인. 한국 도박판 톤 유지 + 가독성.

0. 데이터 시각화 원칙

0-1. 차트 톤

  • ✅ 한국 전통 + 모던 — 차분한 색 (브랜드 Palette)
  • ✅ 단순 + 명확 — 한 차트에 1~2 정보
  • ✅ 위계 명확 — 가장 중요한 숫자 강조
  • ❌ 화려한 색띠 / 그라데이션 폭발
  • ❌ 3D 차트 (가독성 ↓)
  • ❌ 너무 많은 정보 (인지 부담)

0-2. 색 사용 (Palette 일관)

의미 색 용도
이익 / 양수 ACCENT_GOLD #F3C72E 명성치 +, 승률 ↑
손실 / 음수 ACCENT_VERMILION #D93333 명성치 -, 패배
중립 TEXT_SECONDARY #A6A6B2 평균 / 안 플레이
강조 TEXT_GOLD #F2D94D 베스트 / 최고
배경 차트 BG_DARK_BLUE 또는 BG_INK 패널
격자선 TEXT_MUTED + alpha 0.3 옅음

1. DailyActivity 현재 (Sprint 3.15)

1-1. 현재 구현

┌─ 오늘 ────┐ ┌─ 이번 주 ─┐ ┌─ 누적 ───────┐
│  +1,350   │ │  +4,700   │ │  +12,300     │
│  3 런     │ │ 10 런     │ │ 47 런        │
│  1승 2패  │ │ 4승 6패   │ │ 18승 29패    │
└───────────┘ └───────────┘ └──────────────┘

📊 일자별 명성치 변화 (최근 14일)
5-16  │ ███████████░░░ │ +1,350
5-15  │ ██▒▒▒▒▒▒▒▒▒▒▒│  -250
5-14  │ ████████████████ │ +3,600
5-13  │                  │ 안 플레이

구성:

  • 상단 카드 3개 (PanelContainer + VBox)
  • 14일 일자별 (ItemList + 텍스트 막대 █ / ▒)
  • 색: 양수 = 금 / 음수 = 주홍 / 안 플레이 = 회색

1-2. 현재 한계

  • 텍스트 막대 (█/▒) — 정확한 시각 정보 ↓ (몇 px 단위 안 보임)
  • 카드 사이즈 균일 — 정보 위계 부족 (오늘이 가장 중요한지 모름)
  • 추가 정보 부재 (승률 추이 / 단계 분포 등)

2. DailyActivity 강화 (베타 후)

2-1. 상단 카드 — 위계 차등

┌─ 📅 오늘 ────────────────┐ ← 가장 큰 (강조)
│  +1,350                  │   48px Bold
│  3 런 / 1승 2패           │   16px
│  📈 어제 대비 +400       │   12px (delta 표시)
└──────────────────────────┘

┌─ 이번 주 ──┐ ┌─ 누적 ──────┐ ← 작게
│  +4,700    │ │  +12,300    │   32px Bold
│  10 런     │ │  47 런      │   14px
│  4승 6패   │ │  18승 29패  │
└────────────┘ └─────────────┘

변화:

  • 오늘 카드 = 가장 큼 (사용자 최우선 관심)
  • 어제 대비 delta 표시 ("+400" / "-150")
  • 이번 주 / 누적 = 작게 보조

2-2. 차트 — 텍스트 막대 → 정식 막대 차트

📊 일자별 명성치 변화 (최근 14일)

       ▲ +3,600
       ▲
       │
+1,500 │      ▲     ▲
       │      │     │   ▲
       │   ─  │  ─  │  ─│─ ─ ─ 평균선 (점선)
0      ─┼─────────────────────────  zero line
       │  │   │  ▼  │   │
−500   │  ▼      ▒  │
       │
       │
       └────────────────────────────
       5-14 5-15 5-16 5-17 5-18 ...
       (날짜 X-axis, 가독성 위해 일부 라벨)

구성:

  • X-axis: 14일 (5-14 ~ 5-16 형식)
  • Y-axis: 명성치 (자동 스케일)
  • 양수 막대 = 금색 / 음수 막대 = 주홍
  • 평균선 (점선) = TEXT_SECONDARY
  • Zero line = TEXT_PRIMARY 굵음
  • Hover/Tap = 막대 색 진해짐 + 정확한 값 tooltip ("5-16 오늘 +1,350 (3 런)")

Godot 구현:

  • Control._draw() 로 직접 그리기 (다른 자료 카드와 일관)
  • 또는 차트 라이브러리 사용 (Godot 4 호환 차트 plugin)

5. 차트 컴포넌트 spec

5-1. Bar Chart (수평)

사용처: 단계별 통과율, 마패 빈도

스펙:
- 막대 높이: 24~36px
- 막대 간 간격: 4~8px
- 막대 색: 의미에 따라 (브랜드 Palette)
- 라벨: 좌측 (이름), 우측 (값)
- 배경: 옅은 격자선 (10/25/50/75/100% 가이드)

5-2. Bar Chart (수직)

사용처: 14일 명성치 추이, 명성치 분포

스펙:
- 막대 폭: 12~24px
- X-axis 라벨: 회전 0~45도 (공간 부족 시)
- Y-axis 라벨: 좌측 (자동 스케일)
- 양수 / 음수 색 구분
- Zero line 강조 (1px 흰색)

5-3. Line Chart

사용처: 승률 추이, 점수 시간 변화

스펙:
- 라인 굵기: 2~3px
- 노드 (점): 4~6px circle
- 색: 양수 추세 = 금 / 음수 추세 = 주홍 / 평균선 = 회색 점선
- Hover/Tap: 노드 강조 + tooltip

5-4. Heatmap

사용처: 시간대별 플레이, 마패 슬롯 사용 분포

스펙:
- 셀 크기: 32×32 (모바일 24×24)
- 색: 옅음 (alpha 0.1) → 진함 (alpha 0.9)
- 셀 간격: 2~4px
- 색조: 단일 (금 또는 청록) — 강도로 표현

5-5. Stat Card

사용처: 상단 카드 (오늘 / 주 / 누적)

스펙:
- 사이즈: 340×120 (강조) / 240×100 (보조)
- 배경: BG_DARK_BLUE 또는 PanelContainer
- 큰 숫자: 32~48px Bold (위계에 따라)
- 보조: 14~16px Regular
- 색: 양수 = 금 / 음수 = 주홍

6. 모션 (차트)

6-1. 등장 모션

DailyActivity / 통계 화면 진입 시:
1. 상단 카드 fade-in + slide-up (각 0.1s delay, 0.3s)
2. 차트 막대 grow-up (왼쪽부터, 각 0.05s delay, 0.4s, BACK_OUT)
3. 라인 차트 draw-from-left (1.0s linear)

→ 시각 임팩트 + 사용자 시선 흐름 유도.

6-2. Hover / Tap 인터랙션

막대 hover/tap:
- 색 진해짐 (alpha +0.1, 0.1s)
- tooltip 등장 (M-09 토스트 톤, 작은 사이즈)
- 격자선 hover 위치 강조

6-3. 데이터 갱신 모션

새 데이터 들어옴 (라운드 종료 후):
- 막대 / 카드 숫자 카운트업 (0.5s, QUAD_OUT)
- 색 변화 (이익/손실 변경 시 0.3s transition)

9. 데이터 정확성 / 신뢰

9-1. 표시 일관성

  • 모든 숫자 천 단위 콤마 ("1,250", "12,300")
  • 음수 명확 ("-500", "-1,350")
  • 퍼센트 소수점 1자리 ("45.5%", "100.0%")

9-2. 데이터 부족 처리

- 데이터 0건: "기록 없음" 회색 메시지
- 데이터 < 7개: "더 많은 데이터가 필요합니다" 안내
- 안 플레이 날: "안 플레이" (빈 막대)
- 데이터 손실: "데이터 복구 중..." (로딩)

9-3. 신뢰 강화

  • 모든 통계 화면 하단 disclaimer: "본 통계는 본인 로컬 데이터만. Supabase 글로벌 통계는 별도."
  • 데이터 수집 기간 명시 ("최근 30일 / 전체 47 런")

10. 글로벌 통계 (Supabase 연동, 미래)

10-1. 가능한 글로벌 데이터

  • 전체 사용자 단계별 통과율 (자기 vs 평균)
  • 가장 인기 마패 카드 (전 세계)
  • 시드별 통과 기록 (결정성 — 같은 시드 도전 가능)
  • 일일 active users / 신규 가입

10-2. 글로벌 vs 로컬 시각 구분

[자기 통계] — 금색 강조
[글로벌 평균] — 회색 점선

예시:
단계 3 통과율
  자기: ████████ 30% (금)
  글로벌: ████████████ 45% (회색)

→ "내가 평균보다 잘하나 / 못하나" 즉시 인지.

11. 작업 우선순위

Tier 1 — 베타 직전 (이미 구현)

  • DailyActivity 상단 카드 3개
  • 14일 텍스트 막대 차트

Tier 2 — 베타 후

  • DailyActivity 강화 — 오늘 카드 강조 + delta 표시
  • 텍스트 막대 → 정식 막대 차트 (Godot _draw)
  • 탭 분리 (명성치 / 승률 / 단계 / 마패)
  • Hover/Tap tooltip

Tier 3 — 정식 출시 후

  • 통계 화면 신규 (단계별 통과율 / 명성치 분포 / heatmap)
  • 마패 카드 통계 (장착 빈도 / 평균 점수)
  • 글로벌 통계 (Supabase 연동)
  • Steam achievement 통계 통합

13. 결론

데이터 시각화의 목표 = 사용자가 자기 행동을 인지하고 다음 행동 결정.

K-Sutda 의 통계는:
