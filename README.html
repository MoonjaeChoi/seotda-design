각 파일은 마크다운 원본 (.md) + 배포용 HTML (.html) 한 쌍으로 제공됩니다.

  • 마크다운: GitHub / VSCode / Typora 등에서 편집
  • HTML: 브라우저에서 바로 열어 가독성 좋게 확인 (외부 의존 0, 다크모드 자동 대응)

🎯 작업 시작 순서

  1. 01-brief.html 브라우저에서 열기 (또는 .md 를 GitHub 에서 보기)
  2. 게임 톤 + 디자인 시스템 + 9개 화면 우선순위 파악 (5~10분)
  3. 02-asset-spec.html 의 와이어프레임 + 화투 40장 / 마패 67종 사양 확인
  4. 킥오프 미팅 (개발자 ↔ 디자이너) — 톤 합의
  5. 샘플 1~2장 → 톤 확정 → 양산

새 .md 파일 추가하거나 기존 파일 갱신 시:

scripts/tools/md_to_html.sh design-handoff/01-brief.md
# 결과: design-handoff/01-brief.html 생성

요구: node + npx (marked 패키지는 npx 가 자동 캐시).

원칙: .md 와 .html 은 항상 한 쌍. 새 자료 추가 시 .md 작성 → 위 스크립트로 .html 동시 생성 → commit.
